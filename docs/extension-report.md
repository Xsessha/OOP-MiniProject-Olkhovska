# Extension Report

##  Впроваджені покращення
Під час етапу огляду та стабілізації систему було проаналізовано, перевірено та частково покращено для забезпечення відповідності вимогам курсу та стандартам якості архітектури.


## Архітектурні покращення
- **Перевірено суворе розділення шарів:** `Domain - Application - Infrastructure - Presentation`
- **Підтверджено принцип інверсії залежностей (DIP):** Шар Application залежить лише від абстракцій.
- **Забезпечено відсутність "протікання" інфраструктури:** Доменна логіка не містить деталей інфраструктури.
- **Покращено ізоляцію сервісів:** `RentalService` містить усі бізнес-правила.

**Валідація:**
- Інспекція коду
- Інтеграційні тести між шарами

##  Покращення тестування
- **Розширено негативні сценарії:**
  - оренда вже орендованого авто
  - некоректний ввід (GUID, порожнє ім'я)
- **Перевірено повний життєвий цикл оренди:** `оренда - зміна стану - повернення`
- **Забезпечено детерміновану поведінку** основних робочих процесів.

 **Показники якості:**
- Основні сценарії використання (Use Cases) повністю покриті
- Бізнес-правила захищені тестами

##  Огляд шару даних (Data Layer)
- **Перевірено коректність абстракції репозиторію:**
  - `ICarRepository`
  - `IRentalRepository`
- **Підтверджено можливість заміни реалізації сховища:** `InMemory - Файлове`

 **Висновок:** Архітектура підтримує розширення збереження даних без критичних змін (breaking changes).
