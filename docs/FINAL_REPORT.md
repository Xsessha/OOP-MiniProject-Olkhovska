# Підсумковий звіт — Система оренди автомобілів

##  Мета проєкту
Створити багаторівневу, тестовану та розширювану систему оренди автомобілів з використанням принципів ООП та Clean Architecture.
Система демонструє застосування концепцій курсу, включаючи ООП, SOLID, патерни проєктування, тестування та ітеративну розробку.

## Огляд архітектури
Система розділена на:
- **Domain Layer** (Рівень домену / Сутності)
- **Application Layer** (Рівень застосунку / Бізнес-логіка)
- **Infrastructure Layer** (Рівень інфраструктури / Збереження даних)
- **Presentation Layer** (Рівень представлення / Консольний UI)

 **Архітектурні рішення:**
- Бізнес-логіка ізольована від інфраструктури
- Залежності інвертовано через інтерфейси
- Рівень UI не містить бізнес-правил

##  Доменна модель
Основні сутності:
- `Car` (Автомобіль)
- `Customer` (Клієнт)
- `Rental` (Оренда)

 **Зв'язки:**
- Клієнт може мати кілька оренд
- Автомобіль може бути пов'язаний з різними орендами протягом часу
- Оренда пов'язує автомобіль та клієнта


## Реалізовані функції

### Основні сценарії (Use Cases)

**Оренда автомобіля**
- Вибір доступного авто
- Перевірка доступності
- Створення запису про оренду

**Повернення автомобіля**
- Перевірка активної оренди
- Завершення оренди
- Оновлення статусу авто

**Перегляд доступних авто**
- Фільтрація авто за доступністю
- Відображення через UI

### Бізнес-логіка
- Запобігання подвійній оренді
- Валідація вхідних даних
- Відстеження життєвого циклу оренди
- Забезпечення узгодженості стану системи

## Обробка помилок (Error Handling)
Оброблені сценарії:
- Оренда недоступного авто
- Повернення неорендованого авто
- Некоректний ввід користувача
- Відсутні сутності

 **Механізми:**
- Перевірки валідації
- Винятки (Exceptions)

##  Патерни проєктування
- **Repository Pattern** — абстракція над збереженням даних
- **Service Layer Pattern** — інкапсуляція бізнес-логіки

##  Принципи SOLID
- **SRP** — розділення відповідальності між шарами та обробниками
- **DIP** — залежність від абстракцій (інтерфейсів)
- **OCP** — розширювані реалізації репозиторіїв

## Інтеграція концепцій курсу
Проєкт демонструє застосування:
- ООП (класи, інкапсуляція, конструктори)
- Абстракції (інтерфейси)
- Колекції (`List<T>`)
- LINQ (фільтрація, запити до даних)
- Обробка винятків
- Принципи SOLID
- Патерни проєктування
- Багаторівнева архітектура
- Рефакторинг
- Тестування

##  Тестування
- Unit-тести для доменної та сервісної логіки
- Інтеграційні тести для повних робочих процесів
- Покриття негативних сценаріїв

**Quality Gate:**
- Основні процеси повністю протестовані
- Поведінка системи захищена тестами

## Збереження даних (Persistence)
**Поточне:** - збереження в оперативній пам'яті (in-memory)

**Спроєктовано для розширення:**
- Файлове збереження (JSON)
- Підтримка баз даних
