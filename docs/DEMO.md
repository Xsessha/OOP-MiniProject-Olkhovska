# Demo Guide

## Мета демонстрації
Продемонструвати основний функціонал системи, бізнес-правила, обробку помилок та стабільність системи.
## Сценарій 1: Оренда автомобіля (Успішний шлях)

**Кроки:**
- Запустити застосунок
- Обрати опцію 1 (Орендувати автомобіль)
- Ввести ім'я клієнта: John
- Ввести дійсний ID автомобіля (доступного авто)

**Очікуваний результат:**
- Створюється запис про оренду
- Статус автомобіля змінюється на "Орендований" (Rented)
- Відображається повідомлення про успіх

**Що це демонструє:**
- Основну бізнес-логіку
- Зміну стану в системі
- Правильну взаємодію між шарами

## Сценарій 2: Повернення автомобіля (Успішний шлях)

**Кроки:**
- Обрати опцію 2 (Повернути автомобіль)
- Ввести той самий ID автомобіля

**Очікуваний результат:**
- Оренду завершено
- Автомобіль знову стає доступним
- Відображається повідомлення про успіх

**Що це демонструє:**
- Життєвий цикл оренди
- Узгодженість даних
- Коректність бізнес-правил
##  Сценарій 3: Перегляд доступних авто

**Кроки:**
- Обрати опцію 3

 **Очікуваний результат:**
- Відображається список доступних автомобілів
- Орендовані автомобілі виключаються (або показуються окремо)

**Що це демонструє:**
- Фільтрацію даних (LINQ)
- Правильне відображення стану системи

## Сценарій 4: Негативний випадок — Подвійна оренда

**Кроки:**
- Орендувати автомобіль
- Спробувати орендувати той самий автомобіль ще раз
 **Очікуваний результат:**
- Операцію відхилено
- Повідомлення: "Автомобіль вже орендовано" (Car is already rented)

**Що це демонструє:**
- Дотримання бізнес-правил
- Обробку помилок

## Сценарій 5: Некоректний ввід (Invalid Input)

**Кроки:**
- Ввести порожнє ім'я
- Ввести недійсний GUID
- Ввести ID неіснуючого автомобіля
 **Очікуваний результат:**
- Повідомлення про помилку валідації
- Система не "падає" (не завершується аварійно)

 **Що це демонструє:**
- Валідацію вводу
- Надійність (відмовостійкість) системи

## Сценарій 6: Стабільність системи

**Тестування кількох операцій поспіль:**
- Оренда - Повернення - Знову оренда
- Некоректний - Коректний ввід

**Очікуваний результат:**
- Система залишається узгодженою
- Відсутність збоїв або пошкодженого стану
