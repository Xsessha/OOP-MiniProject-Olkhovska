# Self Audit Report


##  Огляд проєкту
Цей проєкт — система оренди автомобілів, побудована з використанням **Clean Architecture** та багаторівневої структури:
`Domain - Application - Infrastructure - Presentation (Console)`

Система реалізує основні робочі процеси, бізнес-правила та стратегію тестування відповідно до вимог навчального курсу.


## Що ЗРОБЛЕНО

###  Архітектура
-  Чітке розділення шарів (Domain, Application, Infrastructure, UI)
-  Інверсія залежностей через інтерфейси
-  Бізнес-логіка повністю ізольована в сервісному шарі (Service Layer)
-  Відсутність прямої залежності домену від інфраструктури
Відповідає архітектурним вимогам

###  Функціональність
Реалізовані ключові Use Cases:
-  Оренда автомобіля (Rent a car)
-  Повернення автомобіля (Return a car)
-  Перегляд доступних автомобілів (View available cars)
 Відповідає мінімальним функціональним вимогам

###  Бізнес-логіка
-  Запобігає подвійній оренді одного авто
-  Валідує вхідні дані користувача
-  Контролює узгоджений життєвий цикл оренди (Rental lifecycle)
 Відповідає вимогам (є логіка поза звичайним CRUD)
###  Тестування
-  Unit-тести для доменного та сервісного шарів
-  Інтеграційні тести для перевірки повних робочих процесів
-  Покрито негативні сценарії помилок
 Відповідає вимогам


## Покриття тем курсу

-  **OOP** — Створено ядро доменної моделі
-  **SOLID** — Чітко застосовані принципи SRP, DIP, OCP
-  **LINQ** — Активно використовується для фільтрації колекцій
-  **Generics** — Застосовано абстракцію репозиторію
-  **Exceptions** — Реалізовано перевірки станів
-  **Patterns** — Задіяні Repository та Service Layer
-  **Testing** — Наявні Unit та Integration тести
-  **UML** — Присутнє в супровідній документації


##  Перевірка відповідності вимогам

-  5 доменних класів
-  3+ сценарії використання (Use Cases)
-  Бізнес-логіка поза межами CRUD
-  Файлове збереження (Відсутнє)
-  Багаторівнева архітектура
-  Принципи SOLID (3+)
-  2+ патерни проєктування
-  Документація


##  Зрілість проєкту

- Проєкт показує ітеративну еволюцію розвитку
- Містить обґрунтовані архітектурні рішення
- Основна поведінка надійно захищена тестами
